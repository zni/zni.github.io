Some Things I'm Currently Working On

xmachine

Context

The idea was to write an emulator of some kind, but I eventually wanted to turn it into a PDP-11 for some reason.

So it's basically PDP-11 machine code, PDP-11 registers, and some PDP-11-like components. It has a disk controller of sorts to read and write to disk images.

Present Date

Working on rewriting all the components to be standalone programs, which interact through the Unibus (provided by UNIX file sockets). Each program will use libunibus to handle its bus interaction. The idea being there's a main thread (stuff the device does), a priority bus thread (handles getting access to the data bus and passing grants/other signals passively), and a data bus thread (handles data input/ouput either as bus master or slave).

After that is done, I'm probably going to focus on targeting a specific PDP-11 processor, ripping out tty/ and implementing a VT100, or something similar, and other stuff I haven't given much thought to yet.

slippy

I remember wanting to write a Scheme interpreter in Rust that implemented the R5RS specification. I can't remember how far I got. I know macros don't exist, which is where it kind of died.

It does support function definitions and some other things. A highlight of its functionality is in the examples directory in the repo. There is a mini-Forth-like interpreter program.

fslib

So this was a continuation, or final completion, of the goal of getfat. That goal was to write a program to read and write to FAT32 images.

The extended goal, which I haven't finished and will probably come back to, is to support multiple file system formats.
